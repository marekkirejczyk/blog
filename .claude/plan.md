# Blog Migration Plan: Medium → Astro Static Site

## Overview
Build a clean, minimal Astro static blog with a white/black Medium-like design, importing all 63 Medium HTML exports.

## Step 1: Scaffold Astro project
- Initialize Astro project in current directory (blog-claude)
- Use the `blog` template or minimal setup with content collections
- Keep `import/` directory as-is (source data)

## Step 2: Build conversion script
- Create a Node.js script (`scripts/convert-posts.js`) to convert Medium HTML exports → Markdown with frontmatter
- Parse each HTML file to extract: title, date, subtitle/description, body content, featured image
- Use `turndown` (HTML→Markdown converter) for the body
- Filter out short comment/reply posts (< 2 paragraphs in body, titles like "In general yes.", "Ok now I understand…", etc.) — roughly ~15 comment-style posts to exclude
- Generate slugs from filenames (strip dates and Medium IDs)
- Output to `src/content/blog/` as `.md` files with frontmatter:
  ```yaml
  ---
  title: "Post Title"
  date: "2024-01-01"
  description: "subtitle from Medium"
  heroImage: "url-to-medium-cdn-image"  # if present
  ---
  ```
- Keep Medium CDN image URLs (they remain accessible) rather than downloading all images

## Step 3: Configure Astro content collections
- Define blog collection schema in `src/content.config.ts`
- Fields: title, date, description, heroImage (optional)

## Step 4: Create layouts and pages
- **BaseLayout.astro** — HTML shell, meta tags, global styles
- **BlogPost.astro** — Single post layout with title, date, content
- **src/pages/index.astro** — Home page with chronologically sorted post list
- **src/pages/blog/[slug].astro** — Dynamic route for individual posts

## Step 5: Design (Medium-like, minimal black & white)
- Max-width 680px content column (like Medium)
- Georgia/serif font for body, sans-serif for headings
- Black text on white background
- Subtle gray for metadata (dates, descriptions)
- Clean typography: large titles, comfortable line-height
- Responsive (mobile-friendly)
- No sidebar, no footer clutter — just content
- Simple header: blog name + optional nav

## Step 6: Build & verify
- Run `astro build` to generate static output in `dist/`
- Verify all posts render correctly
- Test locally with `astro dev`

## Tech Stack
- Astro 5.x
- Content collections (built-in)
- `turndown` for HTML→MD conversion
- Pure CSS (no framework needed for this minimal design)
